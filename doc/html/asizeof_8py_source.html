9 '''
10 This module exposes 9 functions and 2 classes to obtain lengths and
11 sizes of Python objects (for Python 2.2 or later [#test]_).
12 
13 **Public Functions** [#unsafe]_
14 
15  Function **asizeof** calculates the combined (approximate) size
16  of one or several Python objects in bytes. Function **asizesof**
17  returns a tuple containing the (approximate) size in bytes for
18  each given Python object separately. Function **asized** returns
19  for each object an instance of class **Asized** containing all the
20  size information of the object and a tuple with the referents.
21 
22  Functions **basicsize** and **itemsize** return the basic resp. item
23  size of the given object.
24 
25  Function **flatsize** returns the flat size of a Python object in
26  bytes defined as the basic size plus the item size times the
27  length of the given object.
28 
29  Function **leng** returns the length of an object, like standard
30  ``len`` but extended for several types, e.g. the **leng** of a
31  multi-precision int (or long) is the number of ``digits`` [#digit]_.
32  The length of most mutable sequence objects includes an estimate of
33  the over-allocation and therefore, the **leng** value may differ
34  from the standard ``len`` result.
35 
36  Function **refs** returns (a generator for) the referents of the
37  given object, i.e. the objects referenced by the given object.
38 
39  Certain classes are known to be sub-classes of or to behave as
40  dict objects. Function **adict** can be used to install other
41  class objects to be treated like dict.
42 
43 **Public Classes** [#unsafe]_
44 
45  An instance of class **Asized** is returned for each object sized
46  with the **asized** function or method.
47 
48  Class **Asizer** may be used to accumulate the results of several
49  **asizeof** or **asizesof** calls. After creating an **Asizer** instance,
50  use methods **asizeof** and **asizesof** as needed to size any
51  number of additional objects.
52 
53  Call methods **exclude_refs** and/or **exclude_types** to exclude
54  references to resp. instances or types of certain objects.
55  Use one of the **print\_... methods** to report the statistics.
56 
57 **Duplicate Objects**
58 
59  Any duplicate, given objects are sized only once and the size
60  is included in the combined total only once. But functions
61  **asizesof** and **asized** do return a size value resp. an
62  **Asized** instance for each given object, including duplicates.
63 
64 **Definitions** [#arb]_
65 
66  The size of an object is defined as the sum of the flat size
67  of the object plus the sizes of any referents. Referents are
68  visited recursively up to a given limit. However, the size
69  of objects referenced multiple times is included only once.
70 
71  The flat size of an object is defined as the basic size of the
72  object plus the item size times the number of allocated items.
73  The flat size does include the size for the items (references
74  to the referents), but not the referents themselves.
75 
76  The flat size returned by function **flatsize** equals the result
77  of the **asizeof** function with options *code=True*, *ignored=False*,
78  *limit=0* and option *align* set to the same value.
79 
80  The accurate flat size for an object is obtained from function
81  ``sys.getsizeof()`` where available. Otherwise, the length and
82  size of sequence objects as dicts, lists, sets, etc. is based
83  on an estimate for the number of allocated items. As a result,
84  the reported length and size may substantially differ from the
85  actual length and size.
86 
87  The basic and item sizes are obtained from the ``__basicsize__``
88  resp. ``__itemsize__`` attributes of the (type of the) object.
89  Where necessary (e.g. sequence objects), a zero ``__itemsize__``
90  is replaced by the size of a corresponding C type. The basic
91  size (of GC managed objects) objects includes the overhead for
92  Python's garbage collector (GC) as well as the space needed for
93  ``refcounts`` (used only in certain Python builds).
94 
95  Optionally, size values can be aligned to any power of 2 multiple.
96 
97 **Size of (byte)code**
98 
99  The (byte)code size of objects like classes, functions, methods,
100  modules, etc. can be included by setting option *code=True*.
101 
102  Iterators are handled similar to sequences: iterated object(s)
103  are sized like referents, only if the recursion *limit* permits.
104  Also, function ``gc.get_referents()`` must return the referent
105  object of iterators.
106 
107  Generators are sized as (byte)code only, but the generated
108  objects are never sized.
109 
110 **Old- and New-style Classes**
111 
112  All old- and new-style class, instance and type objects, are
113  handled uniformly such that (a) instance objects are distinguished
114  from class objects and (b) instances of different old-style classes
115  can be dealt with separately.
116 
117  Class and type objects are represented as <class ....* def>
118  resp. <type ... def> where an '*' indicates an old-style class
119  and the def suffix marks the definition object. Instances of
120  old-style classes are shown as new-style ones but with an '*'
121  at the end of the name, like <class module.name*>.
122 
123 **Ignored Objects**
124 
125  To avoid excessive sizes, several object types are ignored [#arb]_ by
126  default, e.g. built-in functions, built-in types and classes [#bi]_,
127  function globals and module referents. However, any instances
128  thereof are sized and module objects will be sized when passed
129  as given objects. Ignored object types are included if option
130  *ignored* is set accordingly.
131 
132  In addition, many ``__...__`` attributes of callable objects are
133  ignored [#arb]_, except crucial ones, e.g. class attributes ``__dict__``,
134  ``__doc__``, ``__name__`` and ``__slots__``. For more details, see the
135  type-specific ``_..._refs()`` and ``_len_...()`` functions below.
