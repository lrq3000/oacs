18 class SummaryTracker(object):
19  """ Helper class to track changes between two summaries taken.
20 
21  Detailed information on single objects will be lost, e.g. object size or
22  object id. But often summaries are sufficient to monitor the memory usage
23  over the lifetime of an application.
24 
25  On initialisation, a first summary is taken. Every time `diff` is called,
26  a new summary will be created. Thus, a diff between the new and the last
27  summary can be extracted.
28 
29  Be aware that filtering out previous summaries is time-intensive. You should
30  therefore restrict yourself to the number of summaries you really need.
31 
32  """

34  """Constructor.
35

37  impact on new summaries, iff you decide to exclude them from further
38  summaries. Therefore it is suggested to use them economically.
39 
40  Keyword arguments:

48  """Return a summary.
49 
50  See also the notes on ignore_self in the class as well as the

52

54  if not self.ignore_self:
55  res = summary.summarize(muppy.get_objects())
56  else:
57  # If the user requested the data required to store summaries to be
58  # ignored in the summaries, we need to identify all objects which
59  # are related to each summary stored.
60  # Thus we build a list of all objects used for summary storage as
61  # well as a dictionary which tells us how often an object is
62  # referenced by the summaries.
63  # During this identification process, more objects are referenced,
64  # namely int objects identifying referenced objects as well as the
65  # correspondind count.
66  # For all these objects it will be checked wether they are
67  # referenced from outside the monitor's scope. If not, they will be
68  # subtracted from the snapshot summary, otherwise they are
69  # included (as this indicates that they are relevant to the
70  # application).
71 
72  all_of_them = [] # every single object
73  ref_counter = {} # how often it is referenced; (id(o), o) pairs
74  def store_info(o):
75  all_of_them.append(o)
76  if id(o) in ref_counter:
77  ref_counter[id(o)] += 1
78  else:
79  ref_counter[id(o)] = 1
80 
81  # store infos on every single object related to the summaries
82  store_info(self.summaries)
83  for k, v in self.summaries.items():
84  store_info(k)
85  summary._traverse(v, store_info)
86 
87  # do the summary
88  res = summary.summarize(muppy.get_objects())
89

91  for _id in ref_counter:
92  # referenced in frame, ref_counter, ref_counter.keys()
93  if len(gc.get_referrers(_id)) == (3):
94  summary._subtract(res, _id)
95  for o in all_of_them:
96  # referenced in frame, summary, all_of_them
97  if len(gc.get_referrers(o)) == (ref_counter[id(o)] + 2):

141 class ObjectTracker(object):
142  """
143  Helper class to track changes in the set of existing objects.
144 
145  Each time you invoke a diff with this tracker, the objects which existed
146  during the last invocation are compared with the objects which exist during
147  the current invocation.
148 
149  Please note that in order to do so, strong references to all objects will
150  be stored. This means that none of these objects can be garbage collected.
151  A use case for the ObjectTracker is the monitoring of a state which should
152  be stable, but you see new objects being created nevertheless. With the
153  ObjectTracker you can identify these new objects.
154 
155  """
